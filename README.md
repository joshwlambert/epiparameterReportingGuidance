# Impact of ambiguously reported epidemiological parameters

<!-- badges: end -->

A research compendium of the analysis code used to produce the figures in the
manuscript on **The impact of ambiguously reported epidemiological parameters for infectious disease modelling and recommended best practices**.

Epidemiological analyses are typically chained: the output of one analysis (e.g.
an estimate of the incubation period or onset-to-death delay) becomes the input
to the next (e.g. a case fatality ratio, reproduction number, or quarantine
window). Incomplete or ambiguous reporting of a parameter at one link in this
chain propagates downstream and can change public health conclusions. This
repository accompanies the manuscript by providing fully reproducible scripts for
each of the illustrative case studies.

> **Scope.** This repository exists purely to reproduce the manuscript figures.
> It is organised as an R package so that a few shared helper functions can be
> loaded cleanly and so that the exact software environment can be pinned, but it
> is not intended to be installed from CRAN or used as a general-purpose library.

## Case studies and figures

Each `plot_*.R` script in [`scripts/`](scripts/) is self-contained and writes a
single figure to [`plots/`](plots/):

| Script | Output figure | Case study |
| --- | --- | --- |
| [`plot_ambiguous_distribution_family.R`](scripts/plot_ambiguous_distribution_family.R) | [`plots/ambiguous_distribution_family.png`](plots/ambiguous_distribution_family.png) | Ambiguous reporting of the distribution **family** — gamma, lognormal and Weibull fits matched to the same summary statistics. |
| [`plot_ambiguous_parameterisation.R`](scripts/plot_ambiguous_parameterisation.R) | [`plots/ambiguous_parameterisation.png`](plots/ambiguous_parameterisation.png) | Ambiguous distribution **parameterisation** (shape + scale vs shape + rate) and its effect on a real-time case fatality ratio. |
| [`plot_correlated_parameter_mean_bias.R`](scripts/plot_correlated_parameter_mean_bias.R) | [`plots/correlated_parameter_mean_bias.png`](plots/correlated_parameter_mean_bias.png) | Bias when deriving the distribution mean from **correlated** gamma shape/scale estimates (illustrated with a bespoke MCMC). |
| [`plot_onset_to_death_cfr_bias.R`](scripts/plot_onset_to_death_cfr_bias.R) | [`plots/onset_to_death_cfr_bias.png`](plots/onset_to_death_cfr_bias.png) | Effect of the assumed onset-to-death distribution on case fatality ratio estimates. |
| [`plot_incubation_quantile_uncertainty.R`](scripts/plot_incubation_quantile_uncertainty.R) | [`plots/incubation_quantile_uncertainty.png`](plots/incubation_quantile_uncertainty.png) | Uncertainty in an upper quantile of the incubation period. |

Two further scripts — [`cfr_Anne.R`](scripts/cfr_Anne.R) and
[`est_cfr_alternative.R`](scripts/est_cfr_alternative.R) — contain exploratory
case fatality ratio likelihood code. They are kept for provenance but are **not**
used to generate any manuscript figure.

## Reproducing the figures

The project uses [`renv`](https://rstudio.github.io/renv/) to pin the exact
package versions used in the manuscript.

1. **Clone the repository** and open `epiparameterReportingGuidance.Rproj` in
   RStudio, or set your working directory to the repository root. All scripts use
   paths relative to the root (e.g. `plots/`), so they must be run from there.

2. **Restore the software environment.** From an R session in the project:

   ```r
   # install.packages("renv")   # if renv is not already available
   renv::restore()
   ```

   This installs the package versions recorded in `renv.lock`.

3. **Load the shared helper functions** in `R/` (required by
   `plot_correlated_parameter_mean_bias.R`, harmless for the others):

   ```r
   # install.packages("devtools")  # if needed
   devtools::load_all(".")
   ```

4. **Run a script** to regenerate its figure into `plots/`:

   ```r
   source("scripts/plot_correlated_parameter_mean_bias.R")
   ```

   To regenerate every figure:

   ```r
   for (f in list.files("scripts", pattern = "^plot_.*\\.R$", full.names = TRUE)) {
     source(f)
   }
   ```

Scripts that involve simulation set a random seed where reproducibility of the
exact figure matters.

## Software environment

- **R version:** 4.4.2 (recorded in `renv.lock`)
- **Dependency management:** `renv` (147 packages pinned)
- **Key packages:** the analyses draw on the
  [Epiverse-TRACE](https://epiverse-trace.github.io/) ecosystem, including
  `epiparameter`, `simulist`, `cfr` and `incidence2`, alongside `ggplot2`,
  `tidyr`, `dplyr`, `patchwork` and `cowplot`. Exact versions are in
  `renv.lock`.
